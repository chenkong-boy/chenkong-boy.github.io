问题1 访问人数比较多怎么优化

1. 调整 Tomcat 配置

修改 server.xml 文件中的连接器配置

  • maxThreads：设置线程池的最大线程数，增大这个值可以处理更多的并发请求。
  • minSpareThreads 和 maxSpareThreads：设置保持空闲的最小和最大线程数。
  • connectionTimeout：连接超时时间，以毫秒为单位。
  • compression：启用响应压缩，可以减少传输的数据量，默认压缩文本类型的响应。

2. 增加内存和优化 JVM 参数

修改 catalina.sh 或 catalina.bat 文件中的 JVM 配置

  • -Xms 和 -Xmx：设置 JVM 堆的初始大小和最大大小。
  • -XX:NewSize 和 -XX:MaxNewSize：调整新生代内存大小。
  • -XX:PermSize 和 -XX:MaxPermSize：设置永久代大小（适用于 Java 8 以前版本）。
  • -XX:+UseConcMarkSweepGC：使用 CMS 垃圾回收器，适合低延迟需求的应用。
  • -XX:+CMSParallelRemarkEnabled：优化 CMS 垃圾回收过程。
  • -XX:+UseCMSCompactAtFullCollection：在 Full GC 后压缩内存。

3. 使用负载均衡

在高并发访问场景下，单个 Tomcat 实例可能难以承受，可以使用负载均衡将请求分发到多个 Tomcat 实例。例如：

  • Nginx 或 Apache HTTP Server 配合 mod_jk 或 mod_proxy。
  • HAProxy：专为高可用性设计的负载均衡器。

5. 数据库连接池优化

使用数据库连接池时，调整连接池参数来提高性能，例如 Apache DBCP、HikariCP 等连接池：

6. 启用缓存

  • 页面缓存：使用缓存机制，如 Ehcache 或 Redis，缓存频繁访问的数据。
  • 静态资源缓存：通过 Tomcat 或前置代理服务器（如 Nginx）缓存静态资源。

7. 调整操作系统设置

增加文件描述符限制

  • ulimit：增加文件描述符的最大数量，确保 Tomcat 能够打开更多的文件和网络连接。

调整 TCP 参数

修改 /etc/sysctl.conf 文件：

  • net.core.somaxconn：增加监听队列的最大长度。
  • net.core.netdev_max_backlog：增加每个网络接口接收的数据包队列长度。
  • net.ipv4.tcp_max_syn_backlog：增加 SYN 队列的最大长度。
  • net.ipv4.tcp_tw_reuse：启用 TIME-WAIT 连接重用。
  • net.ipv4.tcp_fin_timeout：减少 TIME-WAIT 状态的持续时间。

8. 监控和调优

使用监控工具如 JavaMelody、VisualVM、JConsole、Prometheus 和 Grafana 监控 Tomcat 性能，及时发现并解决性能瓶颈。
